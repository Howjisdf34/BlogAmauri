¿Qué es exactamente DDL?

El Lenguaje de Definición de Datos (DDL, por sus siglas en inglés) es el pilar de SQL que se encarga del diseño estructural. Si imaginas que una base de datos es un edificio, el DDL no son los muebles ni las personas que viven dentro; es el plano arquitectónico, las paredes y los cimientos que sostienen todo.

El Flujo Estructural: Sus 4 Operaciones

Olvídate de las tablas aburridas. El ciclo de vida de la estructura de tus datos se resume en cuatro acciones consecutivas que transforman el esquema:

1. CREATE (Crear)

Es el punto de partida. Le da vida a un objeto completamente nuevo dentro del sistema, como una tabla o un índice.

2. ALTER (Modificar)

Ideal para cuando los requerimientos cambian. Te permite mutar la estructura existente, como añadir una columna de "teléfono" a tus usuarios.

3. DROP (Eliminar)

La opción definitiva. Borra el objeto del mapa de forma permanente. Desaparece tanto la estructura como lo que había dentro.

4. TRUNCATE (Vaciar)

Una alternativa inteligente. Remueve de golpe todos los datos internos de una tabla, pero te deja el cascarón estructural intacto listo para volver a usarse.

Un Ejemplo Práctico

Llevar el DDL a la realidad es bastante intuitivo. Mira cómo interactúan estos comandos en un entorno real a través de este bloque de código ejecutable:

-- Creamos la estructura inicial
CREATE TABLE productos (
    id INT PRIMARY KEY,
    nombre VARCHAR(80),
    precio DECIMAL(10,2)
);

-- Decidimos agregar una columna nueva sin alterar los datos
ALTER TABLE productos ADD stock INT;

-- Si quisiéramos eliminar el catálogo por completo
DROP TABLE productos;

⚠️ Una regla crítica: El Auto-Commit

A diferencia de otras sentencias de bases de datos, los comandos DDL se guardan de forma inmediata y permanente en la mayoría de los motores (como MySQL). No existe un botón de "deshacer" (Rollback). Una vez ejecutas un DROP, no hay vuelta atrás.
